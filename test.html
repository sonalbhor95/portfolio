ContextRoute

Hybrid dense + BM25 retrieval with intent boosting, reranking, and pipeline trace logging.

ContextRoute combines dense semantic search (MiniLM + FAISS) with lexical BM25 scoring, intent-based source boosting, optional cross-encoder reranking, and full pipeline trace logging. It demonstrates that fusion scoring, intent routing, and reranking each contribute measurable retrieval quality improvements.

Why this matters

Improves retrieval reliability when single-strategy search fails on mixed-intent queries.

Key signal

Hybrid dense + BM25 + rerank alpha control

Big Doc Bot

Code intelligence RAG platform with versioned API, async jobs, and deterministic static analysis merge.

Big Doc Bot ingests entire codebases, chunks source files, generates embeddings with sentence-transformers, and persists per-repository FAISS indexes keyed by a repo fingerprint. An async job system handles long-running analysis, and the output merges probabilistic RAG narrative with deterministic static analysis (Pylint, ESLint) into a structured JSON schema.

Why this matters

Designed for scalable code understanding across large repositories with auditable outputs.

Key signal

FAISS fingerprint indexing + async analysis jobs

Plagiarism Detection for Project Report Using Machine Learning

Niraj Mohabey, Yash Gavanang, Abubakkar Khan, Lavesh Singh Chib, Bhushan Patil

International Journal of Engineering Technology and Management Sciences (IJETMS) · 2023

Proposes a content-protecting plagiarism detection approach using machine learning that achieves the same detection effectiveness as cleartext methods while making content exposure practically impossible. Evaluated on 100 academic papers with varying levels of plagiarism.

Key Contributions

•Content-protecting detection approach matching cleartext effectiveness
•Combined NLP and text similarity analysis for accurate large-scale detection
•Evaluation against traditional methods on 100 academic papers

Methods

•LCS (Longest Common Subsequence) via dynamic programming for structural similarity
•Cosine similarity on TF-IDF vectors for semantic comparison
•ML classification combining multiple similarity signals as features

Results

•F1-score of 0.85 versus 0.75 for traditional methods across 100 academic papers
•Average processing time of 0.5 seconds per document with full LCS and semantic scoring
•Superior accuracy and speed compared to conventional text similarity on obfuscated inputs
